# StageCraft Failure Brief -- StagecraftOps/pace-stagecraft-monorepo

## Failed workflow: CI - Notification Service (.github/workflows/ci-notification-service.yml)

## Root cause (from automated analysis)

The `actions/setup-node@v7` cache setup step fails with 'Some specified paths were not resolved, unable to cache dependencies' because `services/notification/notification-service/package-lock.json` does not exist in the repository. The workflow's `cache-dependency-path` points to this file for npm caching, but it has never been committed (or the service directory itself is absent), so the action cannot hash it and aborts the job before `npm ci` even runs.

## Why this is a code-level issue, not a pipeline config issue

The fix requires adding the missing `package-lock.json` (generated by running `npm install` inside `services/notification/notification-service/` and committing the lockfile) to the repository, not changing any workflow YAML — the workflow is correctly configured and simply cannot find a file that should be part of the application's repository content.
